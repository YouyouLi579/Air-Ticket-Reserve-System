# Air Ticket Reservation System

## Overview
This project is a web-based Air Ticket Reservation System developed for the CSCI-SHU 213 Database course at Shanghai New York University :contentReference[oaicite:0]{index=0}. It simulates a real-world airline booking platform with multiple user roles and core reservation functionalities, supported by a relational database.

## User Roles

### Customers
- Search flights by source, destination, and date  
- Purchase tickets  
- View upcoming and past flights  
- Track spending  

### Booking Agents
- Book tickets for customers  
- Earn and track commissions  
- View sales reports and customer activity  

### Airline Staff
- Create and manage flights  
- Add airplanes and airports  
- Update flight status  
- Generate reports and analytics  
- Manage booking agents and permissions  

## Features
- Flight search and status tracking  
- Ticket booking system  
- Multi-role authentication and session management  
- Reporting for revenue, commissions, and sales  
- Role-based access control  
- Relational database integration  

## System Design
The project follows a standard database system pipeline:
1. ER diagram design  
2. Relational schema and SQL implementation  
3. Web application development  

## Tech Stack
- Backend: Python (Flask)  
- Database: MySQL  
- Frontend: HTML, CSS  

## Security
- Password hashing  
- Session-based authentication  
- Input validation and SQL injection prevention  
- Role-based authorization  

## How to Run
1. Set up the MySQL database using the schema  
2. Configure database connection in the backend  
3. Run the Flask server  
4. Access the application via a browser  

## Notes
This is a simplified airline reservation system focusing on database design and backend functionality, with support for future extensions.
